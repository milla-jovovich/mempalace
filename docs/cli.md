## Installation {#install}

```bash
pip install mempalace
```

From source:

```bash
git clone https://github.com/milla-jovovich/mempalace.git
cd mempalace
pip install -e ".[dev]"    # includes pytest, build, twine
```

## init {#init}

Guided onboarding + AAAK bootstrap. Asks about your people and projects,
generates the wing config.

```bash
mempalace init <dir>
```

Produces `~/.mempalace/wing_config.json` and `~/.mempalace/identity.txt`. Run
once per machine.

## mine {#mine}

Ingest files into the palace. Three modes:

```bash
# Project files — code, docs, notes
mempalace mine <dir>

# Conversation exports — Claude, ChatGPT, Slack, plain text
mempalace mine <dir> --mode convos

# Tag with a wing name
mempalace mine <dir> --mode convos --wing myapp

# Auto-classify into decisions, preferences, milestones, problems, emotional context
mempalace mine <dir> --mode convos --extract general
```

<div class="table-wrap" markdown="1">

| Flag                  | Description                                             |
|-----------------------|---------------------------------------------------------|
| `--mode`              | `projects` (default), `convos`                          |
| `--wing <name>`       | Tag all mined content with a specific wing              |
| `--extract <type>`    | `general` — classifies into 5 memory types              |

</div>

## split {#split}

Some transcript exports concatenate multiple sessions into one huge file.
Split them first:

```bash
mempalace split <dir>                      # split into per-session files
mempalace split <dir> --dry-run            # preview first
mempalace split <dir> --min-sessions 3     # only split files with 3+ sessions
```

## search {#search}

Semantic search across all filed memories. Filter by wing or room:

```bash
mempalace search "query"                          # search everything
mempalace search "query" --wing myapp             # within a wing
mempalace search "query" --room auth-migration    # within a room
```

<div class="table-wrap" markdown="1">

| Flag             | Description                                     |
|------------------|-------------------------------------------------|
| `--wing <name>`  | Restrict to a single wing (+12% recall)         |
| `--room <name>`  | Restrict to a single room (+34% recall)         |
| `--limit <n>`    | Max results to return                           |

</div>

## wake-up {#wake-up}

Load L0 + L1 context (~170 tokens). Paste into your AI's system prompt:

```bash
mempalace wake-up                                 # load L0 + L1 context
mempalace wake-up --wing driftwood                # project-specific
mempalace wake-up > context.txt                   # save to file
```

Output is AAAK-compressed by default — 8x fewer tokens than English. See
[AAAK Dialect]({{ '/aaak' | relative_url }}).

## compress {#compress}

Compress a wing into AAAK — lossless, 30x smaller than English:

```bash
mempalace compress --wing myapp
```

## status {#status}

Palace overview — wings, rooms, drawer counts, disk usage:

```bash
mempalace status
```

## Configuration files {#config}

### `~/.mempalace/config.json`

```json
{
  "palace_path": "/custom/path/to/palace",
  "collection_name": "mempalace_drawers",
  "people_map": {"Kai": "KAI", "Priya": "PRI"}
}
```

### `~/.mempalace/wing_config.json`

Generated by `mempalace init`. Maps your people and projects to wings:

```json
{
  "default_wing": "wing_general",
  "wings": {
    "wing_kai": {
      "type": "person",
      "keywords": ["kai", "kai's"]
    },
    "wing_driftwood": {
      "type": "project",
      "keywords": ["driftwood", "analytics", "saas"]
    }
  }
}
```

### `~/.mempalace/identity.txt`

Plain text. Becomes Layer 0 — loaded every session.
