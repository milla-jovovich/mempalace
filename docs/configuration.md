# Configuration

## Config file locations

| File | Purpose |
|------|---------|
| `~/.mempalace/config.json` | Global configuration |
| `~/.mempalace/identity.txt` | L0 identity text (plain text) |
| `<project>/mempalace.yaml` | Project-specific wing and room definitions |
| `~/.mempalace/people_map.json` | Name variant → canonical name mappings |

`mempalace init` creates the global config directory and files under `~/.mempalace/`. Project-specific wing/room settings live in `<project>/mempalace.yaml`. The config directory permissions are set to `0o700` (owner only).

## config.json

```json
{
  "palace_path": "/home/user/.mempalace/palace",
  "collection_name": "mempalace_drawers",
  "topic_wings": ["emotions", "consciousness", "memory", "technical", "identity", "family", "creative"],
  "hall_keywords": {
    "emotions": ["scared", "afraid", "happy", "sad", "love", "feel"],
    "technical": ["code", "python", "bug", "error", "api", "database"],
    "memory": ["memory", "remember", "forget", "recall", "archive"]
  }
}
```

### Fields

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `palace_path` | string | `~/.mempalace/palace` | ChromaDB storage directory |
| `collection_name` | string | `mempalace_drawers` | ChromaDB collection name |
| `topic_wings` | list | see above | Default topic wing names |
| `hall_keywords` | dict | see above | Keywords that map content to halls |
| `people_map` | dict | `{}` | Name variant mappings (alternative to `people_map.json`) |

## Environment variables

Environment variables take precedence over config file values.

| Variable | Overrides | Description |
|----------|-----------|-------------|
| `MEMPALACE_PALACE_PATH` | `palace_path` | Path to the ChromaDB palace directory |
| `MEMPAL_PALACE_PATH` | `palace_path` | Alias for `MEMPALACE_PALACE_PATH` |
| `MEMPAL_DIR` | (hooks only) | Directory for auto-ingest during hook saves |

## Precedence

Configuration values are resolved in this order (first match wins):

1. **CLI `--palace` flag** (directly overrides the palace path for the invoked command)
2. **Environment variables** (`MEMPALACE_PALACE_PATH`, `MEMPAL_PALACE_PATH`)
3. **Config file** (`~/.mempalace/config.json`)
4. **Defaults** (hardcoded in `config.py`)

Note: most CLI commands pass the resolved `--palace` value directly without exporting an env var. The MCP server (`mempalace mcp run --palace ...` or `python -m mempalace.mcp_server --palace ...`) is the exception — it sets `MEMPALACE_PALACE_PATH` so downstream config reads pick it up.

## identity.txt

Plain text file describing who the AI is. Becomes Layer 0 — loaded every session via `wake-up`.

```
I am Atlas, a personal AI assistant for Alice.
Traits: warm, direct, remembers everything.
People: Alice (creator), Bob (Alice's partner).
Project: A journaling app that helps people process emotions.
```

There is no required format. Write whatever you want the AI to know about itself on startup.

## mempalace.yaml

Generated by `mempalace init` in the project directory. Defines the wing name and detected rooms.

```yaml
wing: myapp
rooms:
  - name: auth
    description: Authentication and authorization
    keywords:
      - auth
  - name: billing
    description: Billing and payments
    keywords:
      - billing
      - payments
```

## people_map.json

Maps name variants to canonical names. Supported by the config loader but not yet consumed during mining — the infrastructure exists for future normalization support.

```json
{
  "kai": "Kai",
  "kai's": "Kai",
  "KAI": "Kai"
}
```

## MCP server configuration

The MCP server accepts a `--palace` flag to override the palace path:

```bash
mempalace mcp run --palace /custom/path/to/palace
```

The knowledge graph database is placed alongside the palace when `--palace` is used, at `<palace_path>/knowledge_graph.sqlite3`. Without `--palace`, it defaults to `~/.mempalace/knowledge_graph.sqlite3`.

## Input validation

All wing names, room names, and entity names are validated by `sanitize_name()`:

- Must be a non-empty string
- Maximum 128 characters
- No path traversal characters (`..`, `/`, `\`)
- No null bytes
- Must match: `^[a-zA-Z0-9][a-zA-Z0-9_ .'-]{0,126}[a-zA-Z0-9]?$`

Content is validated by `sanitize_content()`:

- Must be a non-empty string
- Maximum 100,000 characters
- No null bytes

Invalid input raises `ValueError` (Python API) or returns `{"success": false, "error": "..."}` (MCP).

## Programmatic access

```python
from mempalace.config import MempalaceConfig

config = MempalaceConfig()
config.palace_path        # resolved palace path
config.collection_name    # ChromaDB collection name
config.people_map         # name variant dict
config.topic_wings        # list of topic wing names
config.hall_keywords      # hall → keyword list dict

# Create config directory and default config.json
config.init()

# Save a people map
config.save_people_map({"kai": "Kai", "kai's": "Kai"})
```

Override the config directory (useful for testing):

```python
config = MempalaceConfig(config_dir="/tmp/test_mempalace")
```
